# 待开发功能

## 说话人区分（暂缓）

### 状态

- 当前状态：已形成技术方案，未排入本轮开发。
- 当前版本不下载、不加载说话人模型，不在界面中提供无效开关。
- 开发前必须再次经过用户批准。

### 能力边界

MiMo-V2.5-ASR 能识别多人和重叠对话中的文字，但当前公开接口只返回转写文本，不返回说话人标签、句级时间戳或词级时间戳。因此，说话人区分不能只依赖当前的小米 API 响应。

### 推荐技术方案

1. 继续使用 MiMo ASR 生成主转写文本。
2. 使用本地 `sherpa-onnx` 执行 VAD、说话人分段和说话人嵌入聚类。
3. 通过时间区间将本地说话人结果与结构化转写段合并。
4. 默认输出“发言人 1”、“发言人 2”等匿名标签，不猜测真实姓名。
5. 允许用户重命名、换色、合并或纠正发言人。
6. 重叠说话保留多说话人标记，不强制压缩成单人。

### 预计设置项

- 说话人区分：开/关。
- 说话人数量：自动、2、3、4、5+。
- 分析精度：标准/高精度。
- 模型管理：下载、校验、删除、显示磁盘占用。

设置项只能在模型与实际分析管线完成后上线，不允许先放置纯 UI 开关。

### 开发前验证

- Windows x64 Electron 原生模块打包和签名。
- 模型授权、分发条款与按需下载机制。
- 30/60/120 分钟音频的 CPU、内存、耗时和磁盘占用。
- 中文、英文、方言、远场、重叠说话的测试样本。
- 说话人重命名、合并、纠错与历史记录持久化。

### 已知限制

- 说话人聚类不等于身份识别，无法自动得知真实姓名。
- 声音高度相似、时长过短、背景噪声和重叠对话可能导致分类错误。
- 未经用户校正的说话人标签应显示 Beta 或“自动识别”标识。
- 不使用 AI 根据文本内容猜测说话人，避免将推测当作录音事实。
