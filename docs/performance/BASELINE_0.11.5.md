## 合成转写基准

命令：

```powershell
npm run perf:bench
```

夹具：1,200 个段落，每段包含稳定的中文合成文本。

| 基准 | 均值 | 备注 |
|---|---:|---|
| 搜索 1,200 段 | 0.3051 ms | 656 个样本 |
| SSR 渲染 1,200 段详情页 | 123.30 ms | 3 个样本；最大 289.32 ms |

SSR 不能代替 Electron 中的真实交互测量，但它提供了一个可重复信号：搜索纯函数本身并非当前首要瓶颈，大量段落的组件创建与协调成本明显更高。

## 静态性能证据

- 播放器 `timeupdate` 同时更新播放器和详情页状态。
- 详情页每次播放更新线性扫描全部段落。
- 详情页一次创建全部段落 DOM。
- 单段输入会重建完整 `segments`、`text`、队列结果和历史结果。
- 历史保存会重写完整 JSON。
- ASR 切片在 RPM 等待前完成 Buffer 读取和 Base64 转换。
- CSS 包含 19 处 `backdrop-filter` 和 90 处 `box-shadow`。

## 后续对比规则

每个性能 PR 必须：

1. 在同一开发机运行本基准。
2. 记录构建资源变化。
3. 使用相同 1,200 段夹具。
4. 不把基准墙钟时间作为唯一 CI 门禁。
5. 使用行为测试防止架构回退，并用真实 Electron 性能记录确认体验。
