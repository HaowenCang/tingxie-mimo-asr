# 阶段 4：媒体库扩展性

状态：已完成；PR #8 的 GitHub Windows `test-and-build` 通过（1m34s）。

分支：`codex/perf-media-library`

## 目标

- 让媒体库在 10,000 条资产规模下仍能流畅筛选、滚动和选择。
- 避免文件夹计数、选择判断、详情查找和重复导入退化为平方复杂度。
- 避免每次媒体库操作都重新读取、解析 `index.json`。
- 限制批量复制与 FFprobe 对磁盘、CPU 和正在运行的 ASR 任务的竞争。
- 大批量导入时显示扫描、复制、探测进度，同时限制进度 IPC 数量。

## 已完成实现

### Renderer

- 新增一次遍历生成的媒体库派生索引：
  - `assetById`
  - `transcriptById`
  - `linkedTranscriptIds`
  - `folderCounts`
  - `unfiledCount`
  - 预归一化文件名搜索文本
- 选择状态由数组改为 `Set<string>`。
- 媒体列表使用 `@tanstack/react-virtual`，10,000 条记录只挂载可视行与 8 行 overscan。
- 默认搜索只匹配媒体名称、原名称、扩展名和历史摘要，不读取完整转写正文。
- 新建转写导入后的资产关联使用来源 Map，不再为每个文件扫描整个媒体数组。
- 导入界面展示“扫描 → 复制 → 探测 → 完成”阶段状态。

### 主进程

- 新增 `MediaLibraryStore`：
  - 相同媒体库根目录只读取和解析一次索引。
  - 写入成功后更新内存缓存。
  - 使用临时文件加 rename 原子替换索引。
  - 媒体库根目录变化时显式失效缓存。
- 导入重复检测由逐项 `find` 改为来源签名 Map。
- 文件复制采用有限并行：空闲时 2 路；存在 ASR 任务时降为 1 路。
- FFprobe 仅处理新导入资产；空闲时按 CPU 在 2–6 路之间，存在 ASR 任务时降为 1 路。
- 10,000 文件的进度通知每阶段最多约 101 次，避免 IPC 洪泛。
- 批量删除使用 ID Set。
- 复制失败会清理本批次的 `.part` 和已生成副本，不提交媒体索引。

## 数据安全与兼容性

- 媒体索引 schema 仍为版本 1，没有迁移或改写真实用户媒体数据。
- 本阶段测试全部使用临时目录和开发演示数据。
- 原子索引写入在 Windows 上经过“已有索引再次替换”测试。
- 媒体库根目录切换后不会继续读取旧根目录缓存。
- 复制失败不会留下已登记但不存在的媒体记录。
