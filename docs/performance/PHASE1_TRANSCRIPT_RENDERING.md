# 阶段 1：超长转写渲染与编辑优化结果

> 分支：`codex/perf-transcript-rendering`
>
> 基线：`main` / `88a0e6b`
>
> 日期：2026-07-19

## 完成内容

- 使用 `@tanstack/react-virtual` 对动态高度转写时间轴进行虚拟化，只挂载视口附近的段落。
- 搜索跳转、章节跳转和播放跟随统一通过段落索引定位未挂载的行。
- 播放器继续在本地维护高频播放时间；详情页仅在跨越段落边界时更新活动段落。
- 活动段落定位由线性扫描改为对有序时间轴进行二分查找。
- 段落编辑器改为本地草稿，输入期间只更新当前编辑器；600ms 空闲、失焦或卸载时提交。
- 进度事件改为按文件 ID 在同一动画帧内聚合，避免多文件并行时相互覆盖。
- 稳定详情页的关键回调属性，并隔离播放器波形条的重复协调。

段落级 IPC patch 与单记录持久化依赖阶段 3 的分离式历史存储。本阶段先消除逐按键全树更新；阶段 3 将把提交链路进一步缩小为 `transcriptId + segmentId + patch`。

## 性能结果

测试夹具：1,200 段合成转写，不包含用户私人内容。

| 指标 | 阶段 0 基线 | 阶段 1 | 变化 |
|---|---:|---:|---:|
| 1,200 段详情 SSR 平均耗时 | 123.30 ms | 15.53 ms | -87.4%，约 7.94 倍加速 |
| 搜索 1,200 段平均耗时 | 0.305 ms | 0.357 ms | 基准噪声范围内，阶段 5 再做索引化 |
| 1,200 段初始挂载编辑器 | 1,200 | 小于 100 | DOM 数量有界 |
| 前端主包 | 462.23 kB | 488.25 kB | +26.02 kB |
| 前端主包 gzip | 141.32 kB | 150.09 kB | +8.77 kB |

虚拟列表依赖带来约 8.77 kB gzip 增量；阶段 5 将通过 AI Markdown、设置和媒体库按需加载抵消主包增长。

- 90/90 自动化测试通过，共 23 个测试文件。
