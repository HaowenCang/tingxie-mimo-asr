## 已确认根因

- 媒体库筛选范围和真实文件夹 ID 共用字符串类型，根目录/系统范围可能错误进入 `parentId`。
- 文件夹管理使用悬浮菜单，不符合列表内联操作要求。
- 队列空状态只有固定最小高度，没有填满队列剩余内容区。
- 指定“瞭望 [11]”记录中，同一个 172 字文本块在保存的 `chunkIndex=1` 原始 API 返回内连续出现 3 次；不是 React 重复渲染。
- 名称编辑器是单行 `input`，不能随长名称自动换行增高。
- 媒体资产与无音频历史转写使用两套行、详情和点击逻辑。
- 新建转写导航会清空当前结果，侧栏没有最近打开转写入口。

## 当前实测

- 1424×856 空队列：`.queue-empty` 与 `.queue-list` 的上下左右边界完全一致，页面无横向溢出。
- 长名称：编辑器保持 `rows=1`，在窄详情栏内按内容自动增长到实际 `scrollHeight`，不设置最大高度。
- 文件夹操作：选中文件夹后，新建子文件夹、重命名、移动、删除均在列表内显示，无悬浮菜单。
- 无音频历史：左侧方框选择后使用同一详情栏；点击行正文直接打开转写。
- 快捷返回：切换到媒体库后可从侧栏“已打开”一键返回刚才的转写。
