# 听写交互动效制作方案

最后更新：2026-07-22（Asia/Shanghai）

## 1. 目标与范围

本方案只补齐交互的状态连续性，不改变现有 Liquid Glass 视觉语言、信息架构和业务流程。

目标：

- 用户在点击、切换、展开、增删和等待时，始终能看懂“刚才发生了什么、对象去了哪里、下一步是否可操作”。
- 让导航、文件队列、媒体库、转写详情、播放器、AI 对话和设置弹窗使用同一套动效语言。
- 保持长转写、万级媒体库和窗口拖拽时的性能，不以模糊、阴影或大面积重绘换取动感。
- 完整尊重系统和应用内的“减少动态效果”设置。

本轮不包含：

- 背景粒子、视差、持续漂浮、循环渐变等装饰性动画。
- 对正文做逐字出现、滚动触发入场或持续呼吸，避免干扰阅读和编辑。
- 修改页面布局、配色、组件形态或新增与动效无关的业务组件。
- 给虚拟列表的每次滚动挂载添加入场动画。

## 2. 现状审计

当前并非完全没有动画，已有：

- 导航、按钮、队列行和章节卡的 hover 位移与颜色过渡。
- 上传区拖入态、进度条宽度过渡、加载旋转、状态点脉冲。
- 设置开关滑块、下拉菜单和右键菜单入场。
- 章节跳转后的高亮衰减。
- `prefers-reduced-motion` 和应用内 `reducedMotion` 的全局禁用规则。

主要缺口：

1. 页面和工作区直接替换：新建转写、媒体库、转写详情之间没有承接。
2. 侧栏、AI 对话栏和智能速览直接出现/消失，布局瞬间跳变。
3. 队列增删、状态变化、批量操作栏和搜索结果直接跳变。
4. 弹窗只有静态遮罩，打开、关闭和二级确认没有空间层级。
5. 复制、保存、移动、删除、生成完成等结果反馈大多只换文案或图标。
6. 播放/暂停、静音、选中、展开等图标状态缺少过渡。
7. 媒体拖放只有数据行为，缺少拖拽对象、可放置目标和成功落点反馈。
8. CSS 中大量交互只定义 hover，缺少 `:active`、退出态和跨组件一致的时间曲线。

## 3. 动效设计语言

### 3.1 性格

动效关键词为“轻、稳、贴合、可中断”。玻璃面板像轻质实体，有很小的惯性，但不使用夸张弹跳。内容本身尽量淡入淡出，容器才承担短距离位移。

### 3.2 动效层级

| 层级 | 使用场景 | 时长 | 位移/缩放 | 推荐曲线 |
| --- | --- | ---: | --- | --- |
| 即时反馈 | 按下、选中、图标切换 | 80–120ms | `scale 1 → .97 → 1` | `ease-out` |
| 微交互 | hover、焦点、开关、状态色 | 140–180ms | 0–4px | `cubic-bezier(.2,.8,.2,1)` |
| 内容切换 | Tab、搜索结果、检查器内容 | 160–220ms | 4–8px | `cubic-bezier(.2,.8,.2,1)` |
| 容器变化 | 折叠、批量栏、提示条、列表增删 | 200–260ms | 6–12px | 低回弹 spring |
| 视图切换 | 页面、AI 侧栏、设置弹窗 | 240–320ms | 8–20px | 减速曲线/低回弹 spring |
| 结果强调 | 完成、错误、定位高亮 | 600–1600ms | 单次脉冲或颜色衰减 | `ease-out` |

统一约束：

- 普通交互最长不超过 320ms，不让动画拖延下一次操作。
- 离场比入场快约 20%，例如入场 240ms、离场 180ms。
- 同一批列表最多错开前 4–6 项，每项间隔 24–32ms；大量项目直接就位。
- 不对 `backdrop-filter`、大面积 `filter`、模糊半径和大阴影做逐帧动画。
- 优先动画 `transform` 和 `opacity`；颜色、边框只用于小面积控件。
- 高度未知的内容由布局动画处理，不手写任意 `max-height`。

### 3.3 建议 Token

CSS 与 React 共用同一套语义，不允许组件自行发明时长：

```text
duration.instant = 0.10s
duration.fast    = 0.16s
duration.base    = 0.22s
duration.slow    = 0.30s

ease.standard = cubic-bezier(.2,.8,.2,1)
ease.exit     = cubic-bezier(.4,0,1,1)

spring.snappy = { type: spring, stiffness: 520, damping: 34, mass: .7 }
spring.panel  = { type: spring, stiffness: 360, damping: 36, mass: .9 }

distance.xs = 4px
distance.sm = 8px
distance.md = 16px
scale.press = .97
scale.enter = .985
```

## 4. 技术方案

### 4.1 选型

采用混合实现：

- CSS Transition/Keyframes：按钮 hover/press、焦点、颜色、边框、简单图标旋转和已有 loading。
- Motion for React：条件渲染的入场/离场、列表增删重排、侧栏/面板布局变化、共享选中指示器。
- Web Animations API：只用于需要命令式重放的单次强调，例如复制成功或拖放完成；能用 Motion/CSS 表达时不增加第三套抽象。

不建议以原生 View Transition API 作为主方案。它适合少量整页截图式过渡，但当前应用需要可中断的多面板布局、队列增删和并行状态更新；主流程采用实际 DOM 的布局动画更合适。后续可仅在新建转写与媒体库的低频页面切换中做实验，并保留即时切换回退。

### 4.2 基础设施

新增建议：

```text
src/motion/
  tokens.ts             动效时长、曲线、spring 和位移
  variants.ts           fade、fadeUp、panel、listItem、iconSwap
  MotionProvider.tsx    MotionConfig、LazyMotion、减少动画策略
  MotionPresence.tsx    项目统一的 AnimatePresence 包装
```

在应用根节点接入：

- `LazyMotion + domAnimation`，只加载所需功能。
- `MotionConfig reducedMotion={preferences.reducedMotion ? 'always' : 'user'}`。
- 应用内 `reducedMotion=true` 继续作为严格关闭开关：动画立即完成、平滑滚动关闭。
- 保留现有 CSS 媒体查询，避免非 Motion 动画漏网。

### 4.3 状态与动画分离

- 业务状态仍由现有 React state、IPC 和媒体事件驱动。
- 动画只读取 `open/selected/status/loading` 等状态，不创建第二套业务状态。
- 动画完成回调不得决定数据是否保存、删除或发送；数据操作先发生，视觉离场只是表达结果。
- 只有删除确认弹窗等本来就需要等待用户决定的流程可以阻塞业务。

### 4.4 性能红线

- `.library-virtual-row` 与 `.virtual-timeline-row` 已由虚拟化器使用 `transform` 定位，不得直接改成 Motion layout 元素，否则会争用 transform。
- 媒体库滚动挂载和转写段落滚动挂载不得播放入场动画。
- 新导入媒体如需强调，只动画行内部包装的 `opacity/background-color`，不动画虚拟行位置。
- 播放进度、波形和转写跟随不使用 React spring；继续用帧节流和直接样式更新。
- 拖动侧栏、上传区、媒体库栏和 AI 栏时禁用布局补间；松手后只做不超过 120ms 的吸附/归位。
- 动画期间不恢复已经为拖拽关闭的玻璃模糊，避免 GPU 合成压力。
- AI 流式消息只动画消息容器首次出现，不对每个 token 动画。

## 5. 分场景制作清单

### 5.1 全局按钮与输入控件（P0）

- 所有可点击按钮补齐 `:active`：按下缩至 `.97`，80ms；松开回弹 120ms。
- 纯图标按钮按下只缩放图标，不移动整个工具栏。
- disabled 状态用 120ms 淡化，但禁用后不再响应 hover/press。
- 输入框获得焦点时，聚焦环和边框在 160ms 内出现；错误提示用一次颜色强调，不抖动输入内容。
- Copy → Check、Play → Pause、Volume → Mute、Square → CheckSquare 使用 100–160ms 的缩放淡入淡出。
- Chevron 展开状态统一旋转 90°/180°，不直接替换方向图标造成跳闪。

验收：鼠标、键盘和触控板操作都能得到同等反馈，焦点可见性不依赖动画。

### 5.2 应用壳与导航（P0）

- 新建转写 ↔ 媒体库：旧内容淡出并水平移动 8px，新内容从反方向进入，总时长 220–260ms。
- 打开转写详情：工作区淡出，详情容器淡入上移 8px；保留侧栏位置，不做全屏缩放。
- 侧栏折叠：容器宽度采用布局动画；文字先淡出，图标再归中；展开时顺序相反。
- 导航 active 指示条使用共享 `layoutId` 在项目间移动，避免消失后重现。
- 最近打开项新增/移除使用 `AnimatePresence`；移除后其余项平滑补位。

注意：页面切换期间仍允许再次导航，新操作应中断并接管当前动画。

### 5.3 设置、警告和删除确认（P0）

- 遮罩：0 → 1，140–180ms；关闭 120–150ms。
- 弹窗：`opacity 0 → 1`、`translateY(8px) → 0`、`scale(.985) → 1`，220ms；关闭 160–180ms。
- 设置 Tab：内容区交叉淡化 140–180ms，Tab 下划线共享移动；不让弹窗高度剧烈弹跳。
- Token Plan 警告、文件夹删除确认沿用同一 dialog variant。
- 保存成功消息从原位置淡入并停留，不从屏幕边缘飞入。
- 弹窗打开后立即建立正确焦点，不能等动画结束才可键盘操作。

### 5.4 上传与文件队列（P0）

- 拖入窗口：上传区边框、背景光和图标在 160ms 内增强，主图标上移 4px；波形只播放一轮短促起伏。
- 离开拖入区：120ms 复原，不能残留 dragging 样式。
- 放下文件：上传图标做一次 180ms 的缩放确认；新队列行从 `opacity 0 + y(-6px)` 进入。
- 多文件加入：仅前 6 项使用 28ms stagger，后续项目直接显示。
- 队列行移除：180ms 淡出缩小至 `.985`，其余非虚拟行用 layout 平滑补位。
- 状态 `waiting → preparing → extracting → transcribing → done/partial/error`：状态文字交叉淡化；进度条保持连续，不能回跳。
- 完成：Check 单次弹入并将进度条转绿；错误：错误区域做一次 2px、两段式横向提醒，禁用减少动画时只变色。
- “展开详情”和“回到最新进度”使用高度/淡入动画；用户正在手动浏览队列时不自动抢滚动。

### 5.5 媒体库（P1）

- 文件夹展开：Chevron 旋转，子树淡入并展开；收起时反向，200–240ms。
- active 文件夹背景/指示采用共享移动，不让整个树重新闪烁。
- 拖动文件/文件夹时，被拖对象透明度降至 `.72`；可放置目标显示蓝色内描边和 2px 抬升；非法目标仅显示禁用光标和静态语义色。
- 放下成功后目标文件夹做一次 500ms 背景高亮衰减，并更新计数；失败时保留对象位置并显示现有错误消息。
- 批量操作栏从工具栏下方淡入展开，离场时其余内容平滑回位。
- 选择框图标切换使用缩放淡入，表格选中背景 140ms 过渡。
- 右侧检查器在焦点项目变化时使用 100–140ms 内容交叉淡化，不移动整个栏。
- 导入进度、恢复消息和空状态使用 Presence 动画。
- 右键菜单和子菜单补齐退出动画，并根据 Radix `data-side` 决定 4px 的入场方向。

虚拟列表限制：滚动产生的新行不得入场；排序、过滤和换分组优先让列表整体 120ms 交叉淡化，不做每行重排动画。

### 5.6 转写详情与智能速览（P1）

- 智能速览展开/收起：标题保持固定，内容区使用高度布局 + 透明度，240ms；Chevron 同步旋转。
- 智能速览生成成功：关键词、概要和当前 Tab 内容分三层进入，间隔 35ms，总时长不超过 320ms。
- Tab 切换：active 下划线共享移动，内容交叉淡化 160ms；不左右大幅滑动，避免文本阅读方向被误解。
- 搜索结果面板出现/消失使用高度与透明度；上一项/下一项仍即时定位。
- 章节跳转沿用现有 1.6s 高亮，但首 220ms 增加轻微边框扩散；不移动正文卡。
- 重复内容修复提示条、错误条和撤销状态使用 Presence，完成后一次绿色/蓝色高亮衰减。
- 复制成功只动画按钮内图标和文字，不让工具栏宽度跳动；为“复制”和“已复制”预留相同宽度。

虚拟转写限制：不对段落滚动挂载、播放跟随或搜索跳转增加入场/重排动画。

### 5.7 音频播放器（P1）

- 播放/暂停图标交叉缩放 120ms，播放按钮按下 `.96` 后回弹。
- 前进/后退点击时，相应图标旋转 12° 后复位；不动画整条播放器。
- 波形点选后，播放头可在 100–140ms 内移动到目标；连续播放时保持线性直接更新，不使用 spring。
- 静音切换使用图标交叉淡化；音量滑块继续原生响应，不做延迟跟随。
- 倍速下拉沿用统一菜单动效。
- 跳过静音发生时，播放头直接跳转并让“跳过静音”标签闪亮一次，避免用长距离滑动假装播放经过了静音区。

### 5.8 AI 对话（P0/P1）

- 打开：AI 栏从右侧 16px 淡入，主详情区通过 layout 平滑收窄；关闭反向，240–300ms。
- 窗口或面板拖拽调整宽度时禁用补间，避免指针与边界脱节。
- 用户消息发送后从底部 6px 淡入；助手消息容器只在首个 token 出现时入场，后续流式内容不动画。
- 输入建议被点击时，输入框做一次 100ms 聚焦高亮，不模拟逐字输入。
- typing indicator 保留；停止、重试、重新生成等状态用图标交叉淡化。
- 复制 AI 回答后，复制按钮做一次成功缩放并自动复位。
- 清空对话时消息区整体淡出 140ms，再进入空状态；不逐条播放长串退出动画。

## 6. 减少动画与无障碍

存在两条关闭路径，均必须覆盖 CSS、Motion 和命令式动画：

1. 操作系统 `prefers-reduced-motion: reduce`。
2. 应用设置 `preferences.reducedMotion=true`。

严格模式行为：

- 空间位移、缩放、spring、平滑滚动和列表重排全部关闭。
- 状态立即切换；如需避免闪烁，仅允许不超过 80ms 的透明度变化。
- 加载状态不能只靠旋转表达，必须保留“正在生成/正在导入”等文字。
- 错误不能只靠摇动表达，必须保留颜色、图标和可读错误文案。
- 焦点移动、ARIA live 和键盘操作不得等待动画完成。

## 7. 实施阶段

### Phase 0：动效基线与样例

- 引入 Motion，建立 token、provider、variants 和减少动画映射。
- 统一按钮 press、图标替换、panel/list/dialog variants。
- 在开发演示数据上制作 6 个样例：按钮、弹窗、列表增删、折叠、侧栏、状态切换。
- 用同一台 Windows 设备录制 60fps 基线，确认玻璃背景下无明显掉帧。

交付：动效基础层及短视频/动图评审样例，不改业务流程。

### Phase 1：高频骨架（P0）

- 应用页面切换、侧栏折叠、设置弹窗。
- AI 对话栏开合与消息首帧。
- 上传拖入与文件队列增删/状态变化。
- 完成减少动画全链路。

验收后再进入下一阶段，避免一次性铺满所有组件而失去一致性。

### Phase 2：内容工作区（P1）

- 媒体库文件夹、批量栏、拖放目标、检查器和菜单。
- 智能速览、搜索结果、修复提示和 Tab。
- 播放器图标与跳过静音反馈。

### Phase 3：统一与性能收口

- 清理旧的零散时长和重复 keyframes，全部映射到 token。
- 检查深色、浅色、三种强调色、紧凑布局和不同缩放比例。
- 检查 1080×700、1280×720、1440×920 及窗口连续缩放。
- 检查长转写、万级媒体库、批量导入和 AI 流式输出。
- 完成动效验收清单、性能记录和回归测试。

## 9. 测试与验收

### 9.1 功能回归

- 动画开、关两种模式下，所有按钮、菜单、弹窗、拖放、播放器和编辑行为结果一致。
- 快速连续点击导航、开关 AI 栏、切 Tab 时不出现重复面板、幽灵遮罩或焦点丢失。
- 删除、取消、错误和 IPC 失败时，动画不掩盖真实数据状态。
- Escape、Tab、Enter、Space 和方向键在动画期间保持可用。

### 9.2 视觉验收

- 同一层级组件的方向、距离、时长一致。
- 离场没有突然截断；相邻布局没有先跳再补动画。
- 文本不因 scale 产生明显模糊或拉伸。
- 深色模式下动画中间帧不出现白闪。
- 玻璃面板移动时边缘和背景没有明显撕裂或暗角。

### 9.3 性能指标

- 点击后 100ms 内出现可见反馈。
- 常规转场目标 60fps；开发者工具录制中不出现连续 3 帧以上明显超出 16.7ms。
- 页面/面板转场期间不出现超过 50ms 的主线程长任务。
- 拖动布局边界时指针和边界位置保持同步，不因动效产生尾随。
- 10,000 条媒体库与长转写场景的 DOM 挂载数量不因动效增加。
- 构建后记录 Motion 引入前后的 renderer bundle 差值；若超出约定预算，改用 LazyMotion/拆分到现有懒加载页面。

### 9.4 自动化建议

- Vitest：动效开关映射、variant 选择、动画关闭时的即时状态、组件快速重复操作。
- 浏览器/Electron 集成测试：弹窗退出后 DOM 和遮罩均移除、AI 栏快速开关、队列增删、文件夹展开、焦点恢复。
- 视觉回归：关键动画截取起点/中点/终点，而不是只比较最终静态截图。
- 性能回归：保留现有长转写和大媒体库 demo 参数，增加 `motion=off/on` 对照记录。

## 10. 完成定义

以下条件全部满足才算完成：

- P0 与 P1 清单均有实现或明确记录不实现原因。
- 所有动画使用共享 token，没有散落的任意时长和夸张位移。
- 页面、弹窗、列表和侧栏均有完整入场与离场，不只做 hover。
- 减少动画在系统设置和应用设置中都生效。
- 虚拟列表、播放器进度和拖拽布局没有 transform 冲突或性能倒退。
- 关键路径在 1080×700、1280×720、1440×920 下完成录屏验收。
- 浅色、深色、键盘操作和快速连续操作均无可见缺陷。
- 单元测试、构建、现有性能基准和新增集成检查通过。

## 11. 参考实现依据

- Motion 官方文档：布局动画使用 transform，可处理 React 布局变化和退出状态；`AnimatePresence` 用于组件卸载前的离场动画。
- Motion 官方无障碍建议：通过 `MotionConfig` 和 `useReducedMotion` 统一响应减少动画偏好。
- View Transition API 适合截图式视图过渡，但交互会在过渡过程中受到限制，因此不作为多面板和高频列表交互的主实现。
