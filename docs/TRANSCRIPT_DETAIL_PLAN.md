# 转写详情、智能速览与同步播放完整方案

## 1. 范围与已确认决策

本方案包含：

- Apple Liquid Glass 风格的转写详情页。
- AI 智能速览：关键词、全文概要、章节速览、发言总结、要点回顾和行动项。
- 章节卡片插入对应原文位置。
- 带波形、倍速、音量、前后 5 秒、跳过静音和转写跟随的自定义播放器。
- AI 对话字体大小及其他合理的个性化设置。
- 设置项到真实功能消费者的完整链路审计。

本轮不包含：

- 说话人区分。已记录在 [待开发功能](./PENDING_FEATURES.md)。
- 本地时间戳 ASR、强制对齐或其他时间戳模型。
- 词级高亮。当前只提供段落/句子级的模糊跟随。

## 2. 模糊时间估算

### 2.1 目标

不加载本地时间戳模型，利用已知的 ASR 切片起止时间、切片转写文字和文字在切片中的相对位置，为段落产生可用但不冒充精确的估算时间。

### 2.2 数据前提

转写过程保留每个切片的来源数据：

- `chunkId`
- `physicalStartMs` / `physicalEndMs`
- `logicalStartMs` / `logicalEndMs`
- `overlapWithPreviousMs`
- 原始切片转写文字
- 去重后保留的字符范围
- 请求是否成功及重试次数

展示文本仍可跨切片合并，但不能丢失来源切片关系。

### 2.3 文字权重

将切片文字分成可展示单元，再计算时间比例：

- 中文、日文、韩文字符：每个记 1 个权重。
- 英文和数字：按单词/连续 token 计权重，避免按字母数过度拉长。
- 标点、空格和 Markdown 符号：不计或仅计很小的权重。
- 句号、问号和感叹号作为强边界；分号只作为软标点，不再单独切段。
- 默认累计 3–6 句话、约 120–220 个汉字（英文约 60–110 词）形成一个展示段；过长无标点文本按最大字数产生展示段，但不改写原文。
- 个性化设置提供紧凑、标准和长段落三档，仅影响之后生成的转写，避免重排时覆盖用户对既有原文的手工编辑。

### 2.4 估算公式

对切片中任一文字单元：

```text
estimatedStart = logicalStart
               + logicalDuration * prefixWeight / totalWeight

estimatedEnd   = logicalStart
               + logicalDuration * (prefixWeight + unitWeight) / totalWeight
```

使用逻辑区间而不是带重叠的物理区间，避免上下文重叠使同一段文字获得两个时间。

如已有 FFmpeg `silencedetect` 结果，可将上述比例投影到非静音区间，但不引入任何时间戳模型。没有静音数据时直接使用完整逻辑区间。

### 2.5 跨切片与重叠处理

- 前缀去重的文字不再分配时间。
- 一个展示段跨越两个切片时，起点取最早来源估算，终点取最晚来源估算。
- 失败切片保留完整时间缺口，不将前后文字拉伸覆盖失败区间。
- 对所有结果做单调性校正，确保时间不倒退且不超出媒体时长。

### 2.6 UI 表达与交互

- 估算时间显示为 `≈ 02:29`，不显示为精确时戳。
- 鼠标悬停提示“根据音频切片和文字位置估算”。
- 点击文字时默认定位到估算起点前 500ms，便于用户获得上下文。
- 播放跟随以段落为单位，不做词级高亮。
- 允许用户在播放到正确位置时执行“设为本段起点”，人工校正优先于自动估算并持久化。

### 2.7 限制

- 说话速度变化、长停顿、唱歌和多人重叠对话会增大误差。
- 估算时间适合快速核对和段落导航，不适合字幕级精确同步。
- 当一个切片很长且文字很少时，整个切片的内部定位都应视为低置信度。

## 3. 转写详情页

### 3.1 顶部文件区

- 文件名、媒体时长、语言、转写状态和估算时间说明。
- 搜索、复制、导出、AI 对话、智能速览重新生成。
- 保留现有右侧 AI 对话边栏，允许收起。

### 3.2 智能速览卡片

- 关键词标签。
- 全文概要。
- `章节速览 / 发言总结 / 要点回顾` 切换。
- 编辑、复制、重新生成和生成时间/Provider 信息。
- 始终显示“由 AI 生成”，不将摘要冒充录音原话。

本轮没有说话人标签，因此“发言总结”定义为对录音中的主要表述、论点和结论做内容级总结，不按人归属。未来说话人功能上线后，再升级为每位发言人的总结。

### 3.3 章节速览与原文

- AI 只能返回已存在的 `startSegmentId` / `endSegmentId`，不允许自由生成时间戳。
- 章节卡片放在对应第一个原文段落前。
- 章节卡显示标题、估算时间范围和 2–4 行摘要。
- 章节卡使用淡紫/淡蓝 Liquid Glass，原文卡使用更中性的白色玻璃，形成明确区别。
- 点击章节卡跳转到章节的估算起点。
- 失败切片仍按时间缺口显示，不被 AI 摘要隐藏。

## 4. 自定义音频播放器

### 4.1 播放控件

- 播放/暂停。
- 倒退 5 秒、前进 5 秒。
- 时间滑块与波形点选。
- 当前时间/总时长。
- 倍速档位：`0.75x / 1x / 1.25x / 1.5x / 1.75x / 2x`。
- 音量滑块与静音。
- 跳过静音。
- 自动跟随文字。

### 4.2 波形与静音

- 使用 FFmpeg 生成降采样波形峰值，不在历史 JSON 中保存过大的原始波形数组。
- 静音区间复用 FFmpeg `silencedetect`，不使用本地 AI 模型。
- 对原本可直接上传而没有进行静音检测的短音频，在需要波形/跳过静音时单独执行一次轻量分析。
- 只跳过超过设定阈值的静音，不跳过正常的语气停顿。
- 用户主动拖动、前进/后退和暂停时不自动改写播放位置。

### 4.3 播放与文字联动

- 当前播放时间命中估算区间时，高亮对应段落。
- 无法唯一命中时，选择起点最接近且不晚于当前时间的段落。
- 用户手动滚动后暂停自动跟随，显示“回到当前播放位置”。
- 不宣称为精确字幕对齐。

## 5. AI 智能内容

### 5.1 Provider

- 使用已保存的小米按量、Token Plan 或 OpenAI 兼容 Provider。
- 新增独立的“智能速览 Provider”选择，默认跟随 AI 对话 Provider。
- Token Plan 保留现有警告与确认机制。
- 首次开启自动生成时明确告知可能产生请求费用。

### 5.2 生成结果

- 关键词。
- 全文概要。
- 章节标题、摘要和对应段落 ID。
- 要点回顾。
- 内容级发言总结，不做说话人归属。
- 行动项、负责人和截止时间；原文未说明时标记“未明确”。

### 5.3 长转写

1. 按可用上下文分块生成局部摘要。
2. 使用局部摘要和对应 segment ID 生成全局结构。
3. 校验章节 ID 存在、范围单调且不重叠。
4. 单块失败时局部重试，不删除原始转写。

### 5.4 过期管理

- 分析结果保存转写内容 Hash、Provider、Model 和生成时间。
- 原文编辑后将智能内容标记为“基于旧版转写”，由用户决定是否重新生成。
- 用户手动编辑过的 AI 内容单独标记，重新生成前提示覆盖。

## 6. 设置系统

### 6.1 外观

- 主题：跟随系统/浅色/深色。
- 界面缩放：90% / 100% / 110% / 120%。
- 转写字体大小。
- AI 对话字体大小。
- 内容密度：紧凑/标准/舒适。
- 玻璃强度：低/标准/高。
- 强调色预设。
- 减少动画：跟随系统/开/关。

### 6.2 转写

- 保留现有识别语言、服务入口和自适应并发。
- 显示估算时间：默认开启。
- 点击文字的预留上下文：0 / 0.5 / 1 / 2 秒。

### 6.3 AI

- 保留 Provider、Base URL、Model、上下文、最大输出和系统提示词。
- 不增加温度设置。
- 智能速览 Provider。
- 转写后自动生成智能内容。
- 生成类型：概要、关键词、章节、发言总结、要点、行动项。
- 输出语言：跟随转写/中文/英文。

### 6.4 播放

- 默认倍速。
- 默认音量。
- 前进/后退秒数，默认 5 秒。
- 默认跳过静音。
- 静音跳过最小时长。
- 默认自动跟随转写。

### 6.5 存储

- 媒体策略：引用原文件/缓存播放副本。
- 缓存上限。
- 自动清理时间。
- 查看占用和立即清理。

### 6.6 防止纯 UI 设置

所有设置使用统一、版本化的 `AppPreferences`：

```text
设置控件
  -> 类型和范围校验
  -> 主进程持久化
  -> 功能消费者
  -> 生效验证
```

每个设置必须同时存在：

- 默认值与迁移测试。
- IPC 保存/读取往返测试。
- 对真实消费者的集成测试。
- UI 修改、重启保留和恢复默认的界面测试。

典型验证：

- AI 字体设置必须改变对话消息的计算字号。
- 倍速必须改变 `media.playbackRate`。
- 音量必须改变 `media.volume`。
- 跳过静音必须在测试静音区间改变 `currentTime`。
- 智能 Provider 必须改变实际 Base URL 和 Model。
- 未有功能消费者的设置不允许出现在发布版。

## 7. 媒体与历史数据

- 历史记录保存原媒体路径或缓存副本 ID。
- 原文件移动或删除时提供“重新定位”，原文和 AI 分析仍可查看。
- 通过主进程受控的本地媒体协议为播放器提供 Range 读取，渲染器不接受任意本地路径。
- 视频可在需要时生成独立播放代理音频，不改变上传 ASR 的音质处理。
- 转写正文编辑改为防抖持久化，显示“保存中/已保存/保存失败”。

## 8. 视觉和响应式原则

- 保留当前 Apple Liquid Glass 语言：多层半透明、边缘高光、柔和折射感、低对比阴影。
- 参考页面的信息层级和功能，不直接复制其大面积不透明紫灰色块。
- 章节卡、原文卡、错误切片和 AI 内容使用不同色调和边框语义。
- 所有玻璃组件固定在布局中，不跟随鼠标。
- 宽屏为“原文 + AI 对话”双列；窄窗口将 AI 对话改为抽屉。
- 播放器固定在详情区底部，不覆盖最后一段文字。
- 支持 `prefers-reduced-motion`，深色主题下保证文本和聚焦环对比度。

## 9. 实施顺序

1. **数据模型与迁移**
   - 保留切片来源、媒体引用、估算时间、人工校正和智能分析。
   - 旧历史记录仍可渲染；没有媒体和切片数据时不显示无效播放控件。

2. **设置基础设施**
   - 完成版本化 Schema、IPC、持久化和 CSS 变量。
   - 首先落地 AI 字体大小，并建立设置链路测试模板。

3. **模糊时间估算**
   - 切片来源保留、文字权重、去重映射、单调校正和人工校正。
   - 不引入任何本地时间戳模型。

4. **媒体播放与波形**
   - 安全本地媒体读取、波形、倍速、音量、前后 5 秒和静音跳过。
   - 完成文字点击定位和段落跟随。

5. **AI 智能速览**
   - 结构化输出、长文分层生成、校验、局部重试、过期和手工编辑保护。

6. **Liquid Glass 详情页**
   - 智能速览、章节卡、原文卡、AI 边栏和底部播放器。

7. **设置与功能全链路审计**
   - 逐项验证计算样式、媒体属性、请求地址、持久化和恢复默认。
   - 删除任何没有真实消费者的设置项。

8. **回归测试与安装包**
   - 极长音频、失败切片、文件移动、无 AI Provider、Token Plan 警告、窄窗口和深色主题。
   - 完成测试、构建、界面回归和新版安装包。

## 10. 验收标准

- 每个设置项都有明确的持久化字段、功能消费者和自动化测试。
- AI 字体大小修改后立即生效，重启后保留。
- 点击任一带估算时间的文字段能开始播放，且 UI 明确告知时间为估算值。
- 估算时间单调、不越界，失败切片不被前后文字覆盖。
- 播放、暂停、前后 5 秒、倍速、音量、波形点选和静音跳过有真实媒体状态测试。
- AI 章节只能引用存在的 segment ID，按原文顺序插入。
- 编辑转写后旧智能内容显示过期状态，不被静默覆盖。
- 媒体不存在时可重新定位，不影响原文和已保存的 AI 内容。
- 旧历史记录可正常打开，不会因缺少新字段而崩溃。
